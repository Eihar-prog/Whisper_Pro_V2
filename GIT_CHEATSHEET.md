# 🚀 Git Шпаргалка для Whisper Pro

## 1️⃣ Начало нового проекта (только один раз)

```powershell
git init                          # Инициализация Git в папке
git add .                         # Добавить все файлы в "корзину"
git commit -m "🎯 Initial commit" # Первый снимок состояния
git branch -M main                # Переименовать главную ветку в main
git remote add origin [URL]       # Связать с GitHub
git push -u origin main           # Отправить всё на сервер
```

## 2️⃣ Ежедневная работа (Сохранение локально + Публикация)

Делай это каждый раз, когда закончил кусок работы:

```powershell
git status                        # Посмотреть, что изменилось
git add .                         # Добавить изменения
git commit -m "Описание изменений"# Сохранить версию
git push                          # Отправить на GitHub (бэкап)
```

## 3️⃣ Работа с ветками (Безопасные эксперименты)

**Создать новую ветку:**

```powershell
git checkout -b имя-ветки         # Создать и сразу переключиться
git push -u origin имя-ветки      # (Опционально) Отправить ветку на GitHub
```

**Вернуться в главную ветку:**

```powershell
git checkout main
```

**Слияние (Влить готовую фичу в main):**

1. Переключись в `main`: `git checkout main`
2. Влей изменения: `git merge имя-ветки`
3. Отправь результат: `git push`

## 4️⃣ Спасательный круг (Если что-то пошло не так)

**Отменить изменения в файле (до коммита):**

```powershell
git restore имя_файла             # Вернуть файл к последнему коммиту
```

**Откатиться к предыдущему коммиту (если уже закоммитил ерунду):**

```powershell
git reset --hard HEAD~1           # ⚠️ Осторожно! Удаляет последний коммит безвозвратно
git push --force                  # Если этот коммит уже улетел на GitHub
```

## 5️⃣ Полезные мелочи

| Команда             | Зачем нужна                                            |
| ------------------- | ------------------------------------------------------ |
| `git log --oneline` | Показать историю коммитов кратко                       |
| `git diff`          | Показать, что именно ты менял в коде                   |
| `git branch`        | Посмотреть список всех веток                           |
| `git pull`          | Скачать обновления с GitHub, если работал с другого ПК |

## 6️⃣ Продвинутый откат и «Машина времени» ⏳

**Посмотреть историю и найти нужный момент:**

```powershell
git log --oneline # Скопируй короткий хеш (например, a1b2c3d)
```

**Режим «Наблюдателя» (Безопасно 👀):**
Переключиться на старый коммит, чтобы посмотреть код или запустить старую версию.

```powershell
git checkout a1b2c3d 
# ...смотришь код...
# Чтобы вернуться в настоящее:
git checkout dev 
```

> ⚠️ В этом состоянии ты в «detached HEAD». Не пиши новый код, пока не переключишься обратно на ветку.

**Режим «Достать один файл из прошлого» 📄:**
Если нужно вернуть только один файл, а остальное оставить как есть:

```powershell
git checkout a1b2c3d -- имя_файла.py
```

**Режим «Отмена всего» (Опасно 💥):**
Если всё сломалось и нужно откатить весь проект к состоянию конкретного коммита:

```powershell
git reset --hard a1b2c3d
```
